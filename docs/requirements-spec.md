# 选煤厂组态软件项目需求说明书（Jeecg-Boot 3.9.1）

## 1. 项目概述
### 1.1 背景
现有上位机监控画面存在老旧、扩展性差、兼容性不足、维护成本高、数据交互不流畅等问题，需建设新一代组态软件替换原系统。

### 1.2 建设目标
- 基于 Jeecg-Boot 3.9.1 + Java17 + Vue3 + PostgreSQL 实现稳定可扩展架构。
- 实现上位机画面组态全功能（编辑、发布、版本、导入导出、动态绑定、脚本扩展）。
- 覆盖选煤厂全流程设备监控与控制。
- 建立实时/历史数据采集、存储、分析、报表与审计体系。
- 满足权限、安全、日志、报警与可运维要求。

### 1.3 范围
覆盖：原煤准备、筛分破碎、重介分选、浮选、压滤脱水、煤泥水处理、产品储运、辅助系统。
不覆盖：硬件改造、PLC 程序开发。

## 2. 总体架构
- **前端**：Vue3 + TypeScript + Jeecg 前端风格（新增模块路由/页面，零改动原功能）。
- **后端**：Jeecg-Boot 3.9.1 新增业务模块（coal-config），不修改框架已有功能。
- **数据层**：PostgreSQL（业务主存储）+ Redis（实时缓存）+ RabbitMQ（异步事件）。
- **接入层**：Modbus、OPC UA/DA、TCP/IP、MQTT 统一采集网关。
- **部署层**：Nginx 反向代理，Windows/Linux/工控机兼容。

## 3. 功能需求
### 3.1 组态编辑器
- 拖拽式组件编辑（设备、仪表、阀门、皮带、泵、料仓、流程箭头）。
- 多图层、对齐、缩放、旋转、吸附。
- 模板库、草稿、版本、发布流程。
- 变量绑定（点位绑定）、动态样式映射、脚本表达式。
- 导入导出（JSON 包 + 资源）。

### 3.2 监控运行态
- 100ms~5s 刷新策略，最小 100ms。
- 设备状态（运行/停止/故障/联锁）与关键参数实时展示。
- 远程控制（启停、调速、阀位）双确认与权限校验。
- 物料流向动画与全流程总览。
- 多屏监控、全屏、缩放平移。

### 3.3 数据与分析
- 实时采集：多协议并行采集 + 异常重传补采。
- 历史库：按点位/时间分区存储，支持压缩归档。
- 统计指标：产量、效率、能耗、开机率、故障率。
- 报表：班/日/月报，支持导出。

### 3.4 报警与日志
- 报警等级：一级紧急、二级重要、三级一般。
- 报警流程：触发→弹窗/声光/短信→确认→处置→归档。
- 日志：系统、操作、通信、报警全链路审计。

### 3.5 系统配置
- 设备台账、通信参数、阈值策略、存储周期、主题布局、版本回滚。

## 4. 非功能要求
- 采集延迟 ≤100ms，画面刷新 ≤200ms，报警响应 ≤500ms，切换 ≤1s。
- 支持 ≥50 在线用户，≥30 路画面并发，≥1000 点位采集。
- 历史查询 ≤3s，报表生成 ≤10s。
- 7x24 稳定运行，年故障率 ≤0.5%。

## 5. 安全要求
- RBAC 权限（菜单/按钮/数据域）。
- 控制操作二次确认 + 审批可选。
- API 鉴权、签名、防重放。
- 传输 TLS，敏感数据加密存储。
- 完整审计追踪。
